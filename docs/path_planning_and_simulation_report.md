# 烘干生产线 AGV 路径规划与 Isaac Sim 仿真报告

## 1. 项目目标

本仿真用于展示含水物料烘车在烘干生产线中的自动搬运流程。系统由烘干区、进料区、冷却区、三台 AGV、烘箱、烘车和可视化状态灯组成。仿真的重点不是单个模型展示，而是完整生产节拍下的多工位物流过程：

1. 进料区按固定节拍产生装有湿料的烘车。
2. 北、南两条烘干走廊各由一台 AGV 负责搬运。
3. AGV 将烘车叉起，送入指定烘箱，放下后退出。
4. 烘箱门关闭，LED 从绿色变为红色，表示正在烘干。
5. 烘干结束后，烘箱门再次打开，烘干区 AGV 将烘车取出并送到烘干区与冷却区交界处。
6. 冷却区 AGV 到交界处接车，再将烘车送入冷却区空车位。

当前实现位于 `scripts/create_oven_room_scene.py`。脚本生成 `scenes/oven_room.usd`，再通过 `scripts/launch_process_visualization.py` 在 Isaac Sim GUI 中打开并循环播放。

## 2. 仿真总体方法

当前方案采用“离线生成场景 + USD 时间采样动画”的方式实现流程仿真。脚本先根据生产节拍计算每台烘车、每台 AGV、每个烘箱门和 LED 的关键时间点，然后把这些状态写入 USD 文件的 time samples。Isaac Sim 播放 USD 时间线时，各对象按照预先写入的关键帧运动。

这种方法的优点是流程稳定、可重复、便于展示整体产线逻辑；缺点是它不是实时闭环控制，也没有依赖物理引擎自动求解接触、碰撞和叉车承载关系。因此，当前仿真主要用于方案展示、节拍验证、空间布局检查和路径合理性说明，而不是最终的动力学控制验证。

## 3. 场景布局

### 3.1 坐标系约定

场景使用 Isaac Sim 默认的右手坐标系：

- `X` 轴：东西方向，西侧为进料区，东侧为冷却区。
- `Y` 轴：南北方向，北侧走廊为正 `Y`，南侧走廊为负 `Y`。
- `Z` 轴：竖直方向，地面附近为 `Z=0`。

脚本中设置 `UsdGeom.SetStageMetersPerUnit(stage, 1.0)`，因此模型单位为米。

### 3.2 烘干区

烘干区由四排烘箱组成，每排 7 台烘箱。当前使用的新烘箱模型中，一台 URDF 对应左右两个门，因此在工艺逻辑上可视为两个独立烘箱位。也就是说，虽然几何上是 4 排、每排 7 台烘箱模型，但调度中每排实际包含 14 个可用子烘箱。

主要尺寸如下：

| 参数 | 当前值 |
| --- | ---: |
| 单个烘箱水平尺寸 | `3.5m x 2.25m` |
| 单排烘箱模型数量 | `7` |
| 烘箱列间距 | `0.3m` |
| 四排中间背靠背间距 | `2.0m` |
| 两条走廊宽度 | `4.5m` |
| 烘干区房间尺寸 | 约 `28.3m x 24.0m` |
| 走廊中心线 | `Y=+5.5m` 与 `Y=-5.5m` |

烘箱排布逻辑为：

- 第 1 排与第 2 排之间形成北侧走廊。
- 第 3 排与第 4 排之间形成南侧走廊。
- 第 2 排与第 3 排背对背，中间留 `2.0m` 间隔。
- 两条走廊在东西两端留开口，西侧连接进料区，东侧连接冷却区。

### 3.3 进料区

进料区位于烘干区西侧，是两条走廊向外延伸出的区域。当前进料延伸长度为 `2.0m`。烘车不是从地底升起，而是在进料点按时间可见性出现：在出现时刻之前隐藏，出现时刻之后显示。

湿料烘车按南北走廊轮流出现：

- 第 1 台出现在北侧走廊。
- 第 2 台出现在南侧走廊。
- 第 3 台再次出现在北侧走廊。
- 以此类推。

当前全局出现间隔为 `40s`，因此同一条走廊上相邻两台烘车的出现间隔为 `80s`。

### 3.4 冷却区

冷却区位于烘干区东侧，宽度为 `15.5m`，南北长度与烘干区一致。冷却区地面为黄色，车位区域用绿色标记，AGV 通道用半透明蓝色标记。

当前冷却区车位参数如下：

| 参数 | 当前值 |
| --- | ---: |
| 冷却区宽度 | `15.5m` |
| 车位列数 | `9` |
| 车位行数 | `7` |
| 单个车位尺寸 | `1.0m x 1.5m` |
| 车位总数 | `63` |
| 西侧南北向主通道宽度 | `4.0m` |
| 最东侧车位到东墙预留 | `1.5m` |

冷却区的布局原则是：西侧靠近烘干区的位置保留一条南北向主通道，主通道连通两条走廊入口；每排车位之间布置东西向横向通道；AGV 搬运烘车时优先沿通道行驶，不横向压过车位。

## 4. 工艺调度

### 4.1 任务对象

每台烘车对应一个任务。任务中记录：

- 任务属于北侧走廊还是南侧走廊。
- 烘车出现时间。
- 分配到的烘箱路径与门侧。
- 对应的走廊中心线。
- 交接后要放入的冷却区车位编号。
- 插入、烘干、取出、交接、冷却停车等关键时间点。

当前脚本中 `PIPELINE_TASKS_PER_AISLE = 8`，因此北、南两条走廊各展示 8 台烘车，总计 16 个完整任务。

### 4.2 烘箱使用顺序

烘箱按“从北到南、从东到西”的顺序使用。由于每台烘箱模型有左右两个门，调度时每个模型会拆成两个子烘箱位。排序时先按行号，再按子烘箱的世界坐标 `X` 从东到西排序。

北侧 AGV 使用第 1 排和第 2 排对应的子烘箱序列；南侧 AGV 使用第 3 排和第 4 排对应的子烘箱序列。这样可以保证两条走廊独立运行，并且都遵循从小编号到大编号的使用顺序。

### 4.3 生产节拍

当前节拍为：

| 参数 | 当前值 |
| --- | ---: |
| 烘车全局出现间隔 | `40s` |
| 同一走廊烘车出现间隔 | `80s` |
| 仿真烘干时间 | `320s` |
| 仿真烘干时间含义 | 约等于 8 台烘车出现的时间 |
| 总动画时长 | `1100s` |

实际生产中烘干时间可达 4 小时，但为了在仿真中展示连续工作状态，脚本将烘干时间压缩为 `8 x 40s = 320s`。这样既能看到烘箱占用、LED 变色、任务排队，又不会让仿真长时间处于等待状态。

### 4.4 AGV 调度策略

烘干区有两台 AGV：

- `F4_1000C_top`：负责北侧走廊。
- `F4_1000C_bottom`：负责南侧走廊。

冷却区有一台 AGV：

- `F4_1000C_cooling`：负责从交界处接车并送入冷却车位。

烘干区 AGV 的调度规则为：

1. 如果入口处已有新烘车，则优先去取新烘车。
2. 如果没有新烘车，但有烘干完成的烘车，则去对应烘箱取出。
3. 如果两类任务都未就绪，则等待下一次任务事件。

这样避免了 AGV 在烘干任务结束前无意义等待，也更接近实际产线中“有料先处理、无料再等待”的工作方式。

## 5. 路径规划设计

### 5.1 路径规划约束

AGV 的运动形式按叉车式车辆进行简化，规划时遵循以下约束：

- 只允许直行、后退和转向。
- 不允许纯横向平移。
- 搬运烘车时，烘车始终保持在 AGV 前方固定距离处。
- 抬叉高度为 `0.4m`，不再使用过高的 `2m` 抬升。
- 搬运阶段线速度统一为 `1.0m/s`。
- 90 度转向时间按约 `1s` 处理。

在 USD 动画中，AGV 的位姿由位置 time samples 和 yaw time samples 共同表示。为了避免 270 度绕远旋转，角度插值使用最短角度差，即从当前 yaw 到目标 yaw 只走最小转角。

### 5.2 坐标到 AGV 姿态的换算

脚本使用 `agv_root_for_cart()` 根据烘车目标位置反算 AGV 根节点位置。核心思想是：如果烘车中心点为 `(cart_x, cart_y)`，AGV 朝向为 `yaw`，烘车应保持在 AGV 前方 `AGV_CART_STANDOFF = 1.25m`。因此：

```text
agv_x = cart_x - cos(yaw) * standoff
agv_y = cart_y - sin(yaw) * standoff
```

该关系保证搬运过程中烘车与 AGV 的相对位置保持一致。烘车动画不是单独自由移动，而是由 AGV 位姿和固定前向距离计算出来，从而避免“AGV 与烘车相对滑动”。

### 5.3 进料到烘箱路径

湿料烘车出现在西侧进料点。烘干区 AGV 的入库路径采用近似小写 `r` 形状：

1. AGV 初始位于走廊中部或上一次任务结束位置。
2. AGV 空车沿走廊路径到达进料点。
3. 叉子升高 `0.4m`，将烘车抬起。
4. AGV 先倒退一段距离，为转向预留空间。
5. 沿走廊直行至目标烘箱门前。
6. 转向 90 度，使烘车朝向烘箱内部。
7. AGV 将烘车送入烘箱。
8. 叉子下降，将烘车放在烘箱内。
9. AGV 退出烘箱入口，返回走廊。

该路径的关键点包括：

- `cart_feed`：入口烘车初始位置。
- `cart_retreat`：叉起后先倒退的位置。
- `cart_turn_start`：目标烘箱门前的转向起点。
- `cart_turn_end`：进入烘箱前的转向终点。
- `cart_inside`：烘车最终进入烘箱的位置。

这一路径的设计目的，是避免 AGV 本体直接深入烘箱内部；AGV 只需要把前方烘车推进烘箱，并在放下后退出。

### 5.4 烘箱取出到交界处路径

烘干结束后，烘箱门打开，负责该走廊的 AGV 前往烘箱门口取车。取出路径基本是入库路径的逆过程：

1. AGV 到达烘箱门口。
2. 叉子升高，将烘车从烘箱中抬起。
3. AGV 后退，将烘车带回走廊。
4. AGV 转回东西向走廊。
5. AGV 沿走廊向东行驶至烘干区与冷却区交界处。
6. 烘车前端停在交界处附近，而不是深入冷却区。
7. 叉子下降，烘车落地。
8. 烘干区 AGV 立即向西侧后退，离开交接位置，避免干扰冷却区 AGV。

交界处的目标位置由 `HANDOFF_CART_FRONT_OFFSET_X = 0.75m` 控制，使烘车靠近冷却区入口但不过度进入冷却区。

### 5.5 冷却区取车路径

冷却区 AGV 不在交界处等待，而是在冷却区初始位置或上一次停车任务结束位置待命。只有当烘干区 AGV 放下烘车并离开后，冷却区 AGV 才启动接车。

空车去接车的路径规划为通道约束路径：

1. 若当前位置不在西侧南北向主通道，则先转向并直行回主通道。
2. 沿主通道南北向移动到目标走廊对应的 `Y` 坐标。
3. 转向西侧，到交界处烘车后方。
4. 叉子升高，接走烘车。

这一段避免了冷却区 AGV 为了追求最短距离而斜穿车位区域。

### 5.6 冷却区停车路径

冷却区停车位按编号从小到大使用。当前车位按行优先生成，先从一行中的西侧到东侧，再进入下一行。

冷却区 AGV 搬运烘车后按以下步骤进入车位：

1. 从交界处后退到西侧南北向主通道。
2. 沿主通道行驶到目标车位所在横向通道。
3. 转向东侧，沿横向通道接近目标车位。
4. 在车位前转向，使烘车以南北向姿态进入车位。
5. 将烘车推进车位。
6. 叉子下降，烘车落地。
7. AGV 后退回横向通道，为下一次任务做准备。

车位姿态由目标车位所在行决定。最北侧和最南侧车位使用相邻内侧通道进入；中间行根据车位在南半区或北半区选择上方或下方横向通道进入。这样可以减少穿越车位的情况。

## 6. 动画与仿真实现

### 6.1 USD 时间采样

所有运动对象都通过 USD time samples 实现动画。脚本设置：

```text
FPS = 24
动画总时长 = 1100s
```

写入 USD 时，秒数会乘以 `24` 转换为 time code。AGV 根节点写入平移和 yaw 旋转；叉子写入局部 `Z` 方向平移；烘车写入平移和 yaw 旋转；烘箱门写入沿 `Z` 方向的平移。

### 6.2 烘箱门动画

烘箱门采用 prismatic joint 的可视化等效方式实现。门的闭合位置为 stroke `0`，打开位置为 stroke `2.0m`。每个任务只打开对应子烘箱的门，而不是整台烘箱两个门同时打开。

门动作分为两次：

1. 入库前开门，烘车进入后关门。
2. 烘干结束后开门，烘车取出后关门。

### 6.3 LED 状态显示

每个烘箱模型上方有左右两个 LED，对应两个子烘箱。LED 通过红色和绿色两个小方块的可见性切换实现：

- 绿色：空闲或任务结束。
- 红色：烘干中。

LED 的状态与任务时间绑定。烘车进入并关门后，LED 变红；烘干完成后，LED 变回绿色。

### 6.4 烘车出现方式

烘车的出现不是通过从地下升起实现，而是通过可见性切换实现：

- 出现时刻之前：烘车不可见。
- 出现时刻之后：烘车在进料点直接可见。

这更符合“前序产线把烘车送到进料位置”的语义，也避免了地面穿模造成的误解。

### 6.5 AGV 和烘车相对位置

搬运阶段由 `dense_carried_samples()` 生成高密度采样点。它会在相邻路径关键点之间插值 AGV 位置、AGV yaw 和烘车高度，同时用 `front_cart_position()` 根据 AGV 当前位姿计算烘车位置。

这种方式保证：

- 烘车始终位于 AGV 前方。
- AGV 转向时烘车跟随 AGV 一起改变方向。
- 烘车不会在叉子上主动自转一圈。
- 叉子下降时烘车高度同步下降。

为了处理烘车视觉上前后对称的特点，脚本对烘车 yaw 做了简化，使 `0` 度和 `180` 度不会表现为一次多余自转。


## 9. 当前仿真效果

当前仿真已经能够表达完整生产流程：

- 两条烘干走廊轮流产生湿料烘车。
- 北、南两台烘干区 AGV 分别服务对应走廊。
- 烘箱按设定顺序从小到大使用。
- 每次只打开目标子烘箱的一扇门。
- 对应 LED 在烘干期间变红，完成后变绿。
- 烘干区 AGV 将烘车送到交界处后立即退出。
- 冷却区 AGV 接力运输烘车，并按车位顺序停放。
- 冷却区路径优先使用南北主通道和东西横向通道。

从系统层面看，该仿真能够展示“进料、入炉、烘干、出炉、交接、冷却停车”的完整闭环。

## 10. 局限性与后续改进

当前实现仍有以下局限：

1. **物理接触尚未真实求解**  
   叉车抬起烘车是通过时间采样保持相对位姿实现，不是由碰撞体、关节约束和接触力自然产生。

2. **路径规划为规则路径，不是实时搜索**  
   当前路径由固定关键点和通道规则生成，不包含 A*、Dijkstra、RRT 或多 AGV 动态避障。

3. **AGV 转向为简化模型**  
   当前用 yaw 插值表达转向过程，并未建立真实车辆转弯半径、轮胎约束和底盘控制器。

4. **任务调度为启发式规则**  
   当前策略是“新进料优先，其次处理已完成烘干任务”，没有做全局最优调度。

5. **碰撞检查主要依赖人工观察**  
   当前通过增大通道宽度、减少车位数量和调整路径关键点减少穿模风险，但还没有自动碰撞检测报告。

后续可以从以下方向增强：

- 为 AGV、烘车、烘箱和墙体建立简化碰撞体。
- 将叉车抬升过程改为物理约束或可控 attach/detach 机制。
- 加入车辆运动学模型，使 AGV 转弯符合最小转弯半径。
- 在冷却区引入基于栅格或道路网络的路径搜索算法。
- 添加多 AGV 交通管理，如路口占用、交界处互斥和等待区。
- 输出任务完成时间、设备利用率、车位占用率等统计指标。

## 11. 小结

本部分仿真完成了一个面向课程设计展示的烘干生产线物流系统。场景中包含烘干区、进料区和冷却区三类功能区域，并通过三台 AGV 串联起湿料入炉、烘干、出炉交接和冷却停车的流程。

路径规划采用规则化关键点方法：烘干区 AGV 沿走廊完成取车、倒退、转向、入炉、出炉和交接；冷却区 AGV 沿南北主通道和东西横向通道完成接车与停车。仿真层面使用 USD 时间采样生成连续动画，能够稳定展示设备动作、烘车流转、烘箱门开闭和 LED 状态变化，为后续加入物理接触、车辆运动学和自动避障提供了基础。
